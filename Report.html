ABSTRACT

The Markdown to HTML Converter is a modern web application designed to bridge the gap between rapid technical note-taking and professional presentation.

The project was born out of a real-world necessity during online classes. While Markdown is an excellent format for speed, it lacks immediate visual readability on mobile and Windows platforms without dedicated software. This project solves this problem by providing a drag-and-drop interface that instantly converts raw Markdown files into fully stylized, responsive HTML webpages that mimic the clean aesthetics of Google's Material Design.

The application is built using React for the frontend and Bun as the high-performance JavaScript runtime. It is deployed to AWS S3 for static hosting, utilizing AWS CodePipeline and AWS CodeBuild to automate the build and deployment process directly from GitHub.

INTRODUCTION

In the era of online education, the speed of information capture is critical. Students often find themselves needing to jot down complex technical concepts, code snippets, and mathematical formulas in real-time.

Motivation & Problem Statement

The primary motivation for this project stems from a recurring personal challenge experienced during online classes. Often, instructors would ask us to take quick notes or solve problems on the fly. In these moments, I frequently found myself without a pen and paper ready.

To keep up with the lecture, I resorted to typing notes in Markdown on my computer or mobile phone, as it is the fastest way to format text without lifting fingers from the keyboard. However, I encountered a significant friction point: neither Windows nor standard mobile operating systems support Markdown rendering out of the box. Reading raw Markdown files (.md) later was difficult, and sharing them with non-technical peers or teachers looked unprofessional.

The Solution
To address this, I designed the Markdown to HTML Converter. It is a custom-built utility that takes these raw, hurried notes and transforms them into beautiful, readable documents. I specifically engineered the output to resemble a "Google-like" interface (Material Design 3) to ensure the notes felt modern, clean, and familiar.

PROJECT SCOPE

Objectives:

  • To develop a React-based web interface for seamless file conversion.
  • To create a stylized output template based on Material Design 3 (Google's design language).
  • To automate the deployment of the application using AWS CI/CD services.

Expected Outcomes:

  • A functional web app where users can drag and drop .md files and download .html files.
  • The generated HTML must include navigation sidebars, theme controls, and syntax highlighting.
  • A live URL hosted on AWS S3 that updates automatically when code is pushed to GitHub.

TOPIC DETAILS

Key Concepts:

  • Material Design Integration: The project utilizes @material/web components to achieve the distinct "Google-like" look and feel, ensuring accessibility and responsiveness across phones, tablets, and desktops.
  • Single Page Application (SPA): The app is built using React v19, utilizing hooks for managing file processing state.
  • Runtime Performance: Unlike Node.js, this project uses Bun, a modern toolkit that significantly speeds up package installation and bundling.

Relevance to AWS:

  • Storage: AWS S3 is used to host the static assets (HTML, CSS, JS).
  • CI/CD: AWS CodePipeline orchestrates the flow from source control to production.
  • Compute: AWS CodeBuild provides the environment to install Bun and compile the TypeScript React code.

TOOLS & TECHNOLOGIES USED (SRS)

Cloud Infrastructure (AWS):

  • AWS S3: Static website hosting.
  • AWS CodePipeline: Continuous Delivery.
  • AWS CodeBuild: Managed build service.

Software & Frameworks:

  • Language: TypeScript / TSX.
  • Runtime: Bun v1.3.4 (Fast all-in-one JavaScript runtime).
  • Frontend Library: React v19.
  • UI Design: CSS Variables & Material Design 3 tokens (@material/web).
  • Markdown Parser: Marked.js with Katex extension (for math equations).

PROPOSED SYSTEM

The system is a client-side web application designed for simplicity and speed.

User Interface (The "Google" Look)

The UI mimics the clean, google layout found in most Google websites or Android 13+. It features:

  • A clean, centered "Drop Zone" with Material symbols.
  • A dynamic theme switcher that allows users to adjust the Hue/Color of the document, similar to Android's "Material You" theming system.
  • Light and Dark mode toggles.

Functionality

Upon dropping files, the React engine reads the file content, processes it through the mdToHTML function, injects the result into a pre-designed canvas.html template, and triggers a download.

PROJECT ARCHITECTURE / WORKFLOW

CI/CD Pipeline Workflow:

  1. Development: Code is written in VS Code using TypeScript/React.
  2. Source Control: Code is pushed to a GitHub Repository.
  3. Pipeline Trigger: AWS CodePipeline detects the commit.
  4. Build Phase (AWS CodeBuild):
    • Provisions an Ubuntu container.
    • Installs Bun
    • Runs bun install to fetch dependencies.
    • Runs bun run build to generate the dist/ folder.
  5. Deploy Phase: The contents of dist/ are uploaded to the S3 Bucket.
  6. Access: End users access the website via the S3 Endpoint URL.

PROJECT EXECUTION

Step 1: Environment Setup

  • Initialized the project using bun init to leverage its superior speed over npm.
  • Configured tsconfig.json for strict TypeScript type-checking.

Step 2: Frontend Implementation

  • Core Logic: Implemented converter.ts which uses a template injection method. It takes raw Markdown and inserts it into a {{D3-MARKDOWN}} placeholder within a master HTML template.
  • Styling: Designed style.css to support CSS variables. This allows the output HTML to be fully responsive, scaling perfectly from mobile phones (for studying on the go) to desktop monitors.

Step 3: Cloud Deployment

  • Created an S3 bucket with public read access.
  • Configured a buildspec.yml for AWS CodeBuild to handle the custom Bun installation steps.
  • Linked GitHub to AWS CodePipeline to ensure that any UI tweaks pushed to git are live within minutes.

RESULTS / FINDINGS

  • Solved the Note-Taking Gap: I can now take notes in class using simple text, drag them into the app, and instantly have a professional-looking HTML file to read on my phone while commuting.
  • Performance: The build time using Bun was significantly faster compared to traditional Node.js builds.
  • Aesthetics: The Material Design implementation successfully mimics the high-quality interface of Google apps, making the documentation feel premium.

CONCLUSION

This project successfully transformed a personal academic struggle—taking and reading quick notes—into a robust technical solution. By building a custom converter, I not only solved the issue of viewing Markdown on mobile/Windows but also learned how to deploy modern applications using industry-standard DevOps tools like AWS CodePipeline.

FUTURE SCOPE
